# Daily Metrics Automation (Airflow + ClickHouse)

Структура:
-  daily_dag.py - DAG для работы Airflow
-  result.ipynb - результат работы DAG'а

## Описание проекта

Проект реализует автоматизированный ETL-процесс для расчёта ключевых продуктовых метрик за вчерашний день с последующей загрузкой данных в витрину для визуализации (дашборд).

Автоматизация построена на базе Apache Airflow.  
Источник данных — ClickHouse.

## Цель

Ежедневно рассчитывать основные метрики активности пользователей:

### Feed-метрики:
- Количество просмотров (views)
- Количество лайков (likes)

### Message-метрики:
- Количество отправленных сообщений
- Количество полученных сообщений
- Количество уникальных пользователей, отправивших сообщения
- Количество уникальных пользователей, получивших сообщения

Метрики считаются за **вчерашний день** и агрегируются в разрезе:
- OS
- Возраст
- Пол

Результат сохраняется в таблицу: yesterday_metrics_for_report

---

## Архитектура DAG

DAG состоит из 4 этапов:

1. **Extract**
   - Выгрузка действий из `feed_actions`
   - Выгрузка сообщений из `message_actions`

2. **Transform**
   - Объединение данных по `user_id`
   - Расчёт агрегатов

3. **Aggregation**
   - Группировка по:
     - OS
     - Age
     - Gender

4. **Load**
   - Загрузка итогового датафрейма в ClickHouse

---

## Расписание

DAG запускается ежедневно в: 2:00 по МСК.

Расчёт производится за предыдущий календарный день.
